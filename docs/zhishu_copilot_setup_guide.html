5 分钟把 QBP Copilot 接上智枢

你只需要改一个 .env 文件，再去智枢后台加一个 Tool Server。

最重要的一句话

智枢地址变了，就改这里：

C:\path\to\QBP_MGMT\.env

里面这一行换成你的新智枢地址：

ZHISHU_BASE_URL=http://你的智枢内网IP:端口

本教程只做 4 件事

1. 先在智枢开启 API Key

2. 创建并复制 API Key

3. 粘贴到 QBP 的 .env

4. 在智枢添加 QBP 工具

先准备这 3 个东西

智枢地址

例子：http://127.0.0.1:4173

换环境后，这个地址可能会变。

QBP 地址

本机打开：http://127.0.0.1:5008

给智枢访问：http://你的QBP内网IP:5008

智枢 API Key

等下去智枢里生成一个，复制出来。

这里之前确实有 gap

智枢 / OpenWebUI 的 API Key 不是永远显示的。源码里确认到：

Admin Settings → General → Enable API Keys

这个开关没开时，你去用户 Settings / Account 里也看不到 API Key。

如果你不是 admin，还需要管理员给你的账号开启 API Keys 权限。

如果网页里根本找不到这个开关：

直接用下面的硬解法，在智枢项目根目录创建或修改 .env 文件。

1

先登录智枢

你要做：

打开智枢地址，先登录进去。

智枢地址一般是：

http://127.0.0.1:4173/

我现在只能截到登录页，因为没有你的智枢登录态。你登录后按下面的步骤继续。

2

管理员先打开 Enable API Keys

你要做：

登录智枢后，进入 Admin Settings，找到 General，把 Enable API Keys 打开。

Admin Settings / General
Enable API Keys 必须打开
API Key Endpoint Restrictions 先不用开
如果你看不到 Admin Settings，说明你不是智枢管理员。需要找管理员帮你打开这个开关。
网页里没有这个开关？直接用这个硬解法：

打开智枢项目根目录：

C:\path\to\open-webui

新建或修改 .env 文件，写入：

ENABLE_API_KEYS=True
USER_PERMISSIONS_FEATURES_API_KEYS=True

然后重启智枢。这个项目的后端会读取根目录 .env，重启后 API Key 功能会打开。

3

在用户设置里创建 API Key

你要做：

点左下角头像 / 用户名 → Settings → Account → 找到 API Key → 点 Create new key。

User Settings / Account
API Key 这里会出现 key 输入框
Create new key 点这里 复制按钮

复制出来的 Key 可能只显示一次，先放到临时记事本。

还是看不到 API Key？

按这个顺序查：

1. Admin Settings / General 里的 Enable API Keys 是否打开。

2. 如果网页里没有这个入口，智枢根目录 .env 是否有 ENABLE_API_KEYS=True。

3. 当前账号是不是 admin。

4. 如果不是 admin，管理员是否给了你 API Keys 权限，或者 .env 是否有 USER_PERMISSIONS_FEATURES_API_KEYS=True。

5. 改完后有没有重启智枢，并刷新页面或重新登录。

4

打开 QBP 的 .env 文件

文件位置：

C:\path\to\QBP_MGMT\.env

注意：.env.example 只是模板，QBP 真正读取的是 .env。

如果没有 .env，在 PowerShell 里运行：

cd C:\path\to\QBP_MGMT
copy .env.example .env
你要做：

打开 .env，把下面这段按你的环境改好。

ZHISHU_BASE_URL=http://你的智枢内网IP:4173
ZHISHU_API_KEY=把智枢API_KEY粘贴到这里
COPILOT_DEFAULT_MODEL=
COPILOT_FUNCTION_CALLING=default
COPILOT_TOOL_IDS=server:qbp_meeting_mgmt

QBP_PUBLIC_BASE_URL=http://你的QBP内网IP:5008
QBP_TOOL_SERVER_TOKEN=qbp-tool-随便写一串长密钥
我到底要改哪几行？

只看这三行就够了：

ZHISHU_BASE_URL=http://你的智枢内网IP:4173
ZHISHU_API_KEY=你的智枢API_KEY
QBP_PUBLIC_BASE_URL=http://你的QBP内网IP:5008

其他行不懂就先别动。

5

重启 QBP

你要做：

保存 .env 后，回到 QBP 的 PowerShell 窗口，按 Ctrl + C 停掉，再重新启动。

cd C:\path\to\QBP_MGMT
python start.py
打开 QBP 右下角 Copilot，如果不再显示“待配置”，这一步就基本成功了。
6

在智枢里添加 QBP 工具

这一步让智枢能读取 QBP 会议、Topic、会议纪要。

你要做：

智枢后台 → Admin / Settings / Integrations / Tools → 新增 OpenAPI Tool Server。

Name: QBP Meeting MGMT
Server ID: qbp_meeting_mgmt
OpenAPI URL: http://你的QBP内网IP:5008/copilot/tools/openapi.json
Auth Type: Bearer Token
Bearer Token: 填 QBP_TOOL_SERVER_TOKEN 那一串
别填错：

Server ID 必须是 qbp_meeting_mgmt。

Bearer Token 必须和 .env 里的 QBP_TOOL_SERVER_TOKEN 一模一样。

最后这样验收

在 QBP 验证

打开 QBP → 登录 → 点右下角 Copilot → 输入 @。

能弹出会议列表 = QBP 这边正常。

问一句试试

@CM20260001 总结这个会议的采购风险

能回复 = QBP → 智枢 通了。

用接口验证状态

登录 QBP 后打开：

http://127.0.0.1:5008/copilot/status

看到下面这样就对：

"api_key_configured": true
"tool_server_configured": true
"tool_ids": ["server:qbp_meeting_mgmt"]

换环境时只看这里

如果你的智枢地址不是 http://127.0.0.1:4173，不用改代码，只改 .env。

智枢地址变了 
ZHISHU_BASE_URL=http://新的智枢内网IP:端口
QBP 地址变了 
QBP_PUBLIC_BASE_URL=http://新的QBP内网IP:5008

然后智枢后台的 Tool Server URL 也跟着改：

http://新的QBP内网IP:5008/copilot/tools/openapi.json
改完地址后，一定要重启 QBP。
